Identifier les types Schema.org adaptés

Chaque page doit être enrichie avec des données structurées appropriées pour améliorer le référencement SEO et obtenir des rich snippets dans les résultats de recherche. Voici quelques exemples courants :

  • Brand : Utilisé pour baliser une marque ou une organisation.
  • Product : Idéal pour les pages présentant des produits, permettant d'afficher des informations comme le prix, la disponibilité et les avis.
  • Event : Parfait pour les événements tels que des concerts, conférences, avec des détails sur la date, le lieu et les billets.
  • Organization : Sert à décrire une entreprise ou une organisation, incluant des informations comme le nom, l'adresse et le logo.
  • Article : Conçu pour les articles de blog ou de presse, aidant à afficher le titre, l'auteur et la date de publication.
  • LocalBusiness : Adapté pour un commerce local avec une adresse physique, incluant des détails comme les heures d'ouverture, l'adresse et le numéro de téléphone.

Pour explorer l'ensemble des types disponibles et choisir ceux qui conviennent le mieux à votre contenu, consultez la documentation officielle sur Schema.org.

Valider vos données structurées

Une fois ajoutées, validez vos balisages avec :

  • Rich Results Test : Vérifie si vos données permettent des résultats enrichis.
  • Schema Markup Validator : Outil officiel de Schema.org pour détecter les erreurs ou avertissements dans votre JSON-LD.
